Azert Pipeline

A simple synchronous pipeline utilising the pipes and filters pattern.

Welcome to the Azert Pipeline page!

This is a little project I put together very quickly when mocking up a new architecture for a system I was working. After reading various articles I decided the Pipes and Filters pattern was the best approach to take as it fitted very nicely with what I was trying to achieve.

After reading various code examples I decided to try my hand at writing a very lightweight and simple Pipeline that I could use in my projects going forward, this is the result of that.

It is based roughly on this explaination from Microsoft Pipes and Filters Pattern

Simple

I did not need an ayncshronous or parallel pipeline so this is simply a synchronous pipeline, as in, it starts at the start and finishes at the end and processes each filter in turn.

There are very few methods to be aware of, in fact only two:

Pipeline.Register()
Pipeline.Execute()

Extensible

To add a new filter simply implement IFilter and register with your Pipeline:

public class Lowercase : IFilter {
    public int Order => 1;

    public object Process(object input) {

        Console.WriteLine(input.ToString().ToLowerInvariant());

        return input;
    }
}

And register:

pipeline.Register(new Lowercase());

Then start the execution of the pipeline:

pipeline.Execute("test");

The example here is really simple and just passes a string object, you of course can pass any object and then cast it to the type you need in your filter:

public class Lowercase : IFilter {
    public int Order => 1;

    public object Process(object input) {
        var yourType = input as YourType;

        // Some actions here

        return yourType;
    }
}

Future

One next step I will look at is adding a load balancer, this should allow a long running filter to have more than one implementation to speed up the pipeline.
